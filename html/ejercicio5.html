Ejercicio 5: Codificación de vídeo mediante un CDN

El proveedor elegido es Google Cloud for media and entertainment. Se trata de una serie de servicios y herramientas en la nube ofrecida por Google Cloud Platform (GCP) diseñada para procesar, almacenar, codificar y distribuir contenido multimedia de vídeo y audio a gran escala.

Entre las soluciones destacadas se encuentran el Cloud Transcoder API para trabajos de codificación, el Cloud Video Intelligence API para análisis y etiquetado del contenido, o servicios como Cloud Storage y Cloud CDN/Media CDN que permiten almacenar y distribuir contenido con alta disponibilidad y bajo latencia.

A continuación, se detallan sus características y funcionalidades principales:

  • Cloud Transcoder API: Es el servicio principal para codificar contenido multimedia a múltiples formatos y resoluciones, generando resultados optimizados para distintos dispositivos y condiciones de red. Soporta codecs modernos como H.264 y H.265, así como protocolos que permiten streaming en tiempo real, como MPEG-DASH o HTTP Live Streaming (HLS). Permite configurar multitud de opciones de codificación, como la tasa de bits, resoluciones, estructuras GOP, inserción de fotogramas clave, etc.
  • Cloud Video Intelligence API: Este servicio permite analizar el contenido de los vídeos de forma automatizada y posibilita: la identificación de objetos, lugares y acciones mediante algoritmos y modelos de aprendizaje automáticos entrenados; la extracción de metadatos enriquecidos a nivel de vídeo o fotograma; la detección de palabras clave y generación de transcripciones, etc.
  • Cloud Storage: Es el servicio para almacenar datos no estructurados y ficheros multimedia.
  • Cloud CDN/Media CDN: Media CDN es la solución para la entrega específica de contenidos multimedia, que complemente Cloud CDN, la solución para acelerar la entrega de contenido al usuario. Utiliza la infraestructura de Youtube para acercar los videos bajo demando y en directo a los usuarios para acelerar su distribución.
  • Cloud Speech-to-Text y Cloud Translation: Son servicios que permiten generar transcripciones automáticas de vídeos/audios y generar traducciones.
  • Seguridad: Ofrece seguridad del contenido mediante varias capas, tales como poder configurar que los servicios en la nube sigan las prácticas recomendadas de la Asociación Cinematográfica de Estados Unidos (MPA), evaluaciones trimestrales de seguridad por parte de Independent Security Evaluators (ISE), priorizar la seguridad y privacidad de los datos personales de los clientes mediante la aplicación del GDPR, etc.
  • Integración: La integración de Google Cloud for media and entertainment con el ecosistema de Gogle Cloud Platform es nativa y permite integrar y automatizar otros servicios además de los mencionados, tales como pipelines de procesamiento con Cloud Run Functions, integración de flujos de trabajo con herramientas de gestión y orquestación, integración con herramientas de análisis de datos, etc.

Muchas de estas funcionalidades también están disponibles en otros proveedores, tales como servicios de codificación de vídeos, soluciones de distribución y entrega mediante CDN o la facilidad de integración con el ecosistema del proveedor.

Es habitual que, al igual que en Google Cloud for media and entertainment, los proveedores tengan un modelo de coste por consumo.

Por ejemplo, los costes mínimos de Google Cloud que habría que considerar para transformar y entregar archivos multimedia serían:

  • Cloud Transcoder API: El precio varía en función de la duración del vídeo y el tipo de salida (resolución, calidad, etc.). Además, algunas opciones avanzadas pueden tener un coste adicional, tales como codificación en dos pasadas, perfiles específicos, etc.
  • Cloud Storage: El precio depende de la clase de almacenamiento seleccionado y la cantidad de GB almacenados.
  • Cloud CDN/Media CDN: El precio depende de la cantidad de datos que se cachean y se distribuyen a partir de ellas.

Aunque la plataforma de Google Cloud es robusto, fiable y escalable, hay que tener en cuenta ciertas limitaciones y obstáculos:

  • Restricciones en configuración: Google Cloud se orienta a simplificar los procesos para cubrir los casos más comunes, por lo que es posible que algunas configuraciones más complejas relacionadas con la estructura GOP, tasa de bits o perfiles y niveles estén restringidos.
  • Complejidad: La configuración de automatizaciones para procesar videos puede ser compleja y su curva de aprendizaje puede ser alta, ya que se compone de varios servicios.
  • Costes variables: Al tratarse de un modelo de coste por consumo, dificulta realizar estimaciones precisas de gastos y es necesario monitorizar el consumo mensual para controlar los costes.
